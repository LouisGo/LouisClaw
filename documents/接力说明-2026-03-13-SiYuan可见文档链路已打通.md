## 6）边界说明（防止误操作）

- Git 会同步项目代码、skills、文档，以及像 `data/state/siyuan-map.json` 这类派生 state 文件；但这些 state 的内容仍然带有机器 / 实例相关性，不应误认为跨机器天然有效。
- 不要把项目 `.env` 提交进仓库。
- 不要尝试直接写 `.sy` 文件。
- 现在如果目标是“让内容在思源树里可见”，应优先走官方 API，而不是继续尝试把 Markdown 扔进工作空间目录。
- LouisClaw repo 仍然是 source of truth；SiYuan 只是下游消费面。
- 当前已经真实存在的 `AI-Flow` notebook 是**这台机器上的状态**，换机器后不能默认存在。

## 7）快速复现清单（新机器最小接力）

1. `git pull`
2. `npm install`
3. `npm run build`
4. 确认 OpenClaw workspace 指向当前仓库
5. 确认本机 SiYuan 正在运行，kernel 可达：`http://127.0.0.1:6806`
6. 确认本机 SiYuan API token
7. 在项目根目录准备 `.env`（至少含模型/分类配置）
8. 执行：

```bash

SIYUAN_EXPORT_DRIVER=api \

SIYUAN_API_TOKEN=... \
```

9. 在思源里检查是否出现 notebook：`AI-Flow`
